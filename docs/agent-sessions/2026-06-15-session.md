## Goal
Remove the legacy unused `Translate.cs` static class so the configurable `AppSettings.providers.ollama.baseUrl` becomes the single source of truth for the Ollama host URL (and a stray hardcoded `localhost:11434` default / `OLLAMA_BASE_URL` env-var path is eliminated).

## Implementation Details

### Discovery
- `AppSettings.cs:99-104` already defaults `providers.ollama.baseUrl = "http://localhost:11434"` and exposes it through `GetProviderConfig("ollama").BaseUrl` (also convenience accessor `OllamaBaseUrl` at `:153-158`).
- `OllamaOCRProvider` (`OllamaOCRProvider.cs:25-35`) and `OllamaTranslationProvider` (`OllamaTranslationProvider.cs:21-31`) both have a `BaseUrl` property whose setter updates `HttpClient.BaseAddress` live.
- `ProviderFactory` pushes `config.BaseUrl` into both providers (`ProviderFactory.cs:65`, `:107`).
- The Settings window has Base URL text boxes for both OCR Ollama and Translation Ollama panels (`SettingsWindow.xaml:107-109` and `:277-279`), loaded from `config.BaseUrl` (`SettingsWindow.xaml.cs:117`, `:223`), collected on save (`:445`, `:490`), and persisted via `AppSettings.Save()`.
- Test Connection buttons (`TestOcrConnectionButton` / `TestTranslationConnectionButton` in `SettingsWindow.xaml:210`, `:380`) call `CollectOcrSettings()` / `CollectTranslationSettings()` first, so the just-typed Base URL is what gets tested via `/api/tags`.

### Why the deletion is safe
- The only place left in the codebase that read the Ollama host from anywhere other than `AppSettings` was the static constructor of `Translate` (`Translate.cs:22`). With that class gone, every call site for Ollama goes through the provider, and every provider gets its `BaseUrl` from `AppSettings.providers.ollama.baseUrl`.
